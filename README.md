## 为什么选择 KMblog？

### 告别命令行，拥抱可视化

还在为 Hexo 的命令行操作而烦恼吗？还在手动编辑 YAML 配置文件吗？

**KMblog 带来全新体验：**

- **桌面管理工具** - 精美的 GUI 界面，像使用桌面应用一样管理你的博客
- **拖拽式管理** - 文章分类、合集整理，只需拖拽即可完成
- **实时预览** - 所见即所得，编辑和预览同步进行
- **一键部署** - 无需记忆复杂的 Git 命令，点击按钮即可发布

### 不仅仅是静态博客

传统静态博客框架只提供基础的文章展示功能，KMblog 为你准备了更多：

**内置 Monaco 编辑器**
- 在浏览器中直接编辑文章，无需切换工具
- 完整的代码高亮和智能提示
- 支持实时预览和自动保存

**强大的内容组织**
- 合集系统：将系列文章组织成专题，方便读者系统学习
- 加密文章：保护私密内容，只对特定读者开放
- 多维度分类：标签、分类、归档，多种方式组织内容

**现代化主题系统**
- 明暗模式自动切换，跟随系统设置
- 多套预设主题，一键切换风格
- 平滑过渡动画，细节处处用心

### 从 Hexo 迁移？轻而易举

已经在使用 Hexo？没问题！

- **一键迁移工具** - 自动转换 Hexo 文章格式
- **保留所有内容** - 标签、分类、Front Matter 完整保留
- **智能适配** - 自动添加 KMblog 特有的增强字段

### 开发者友好

不同于其他博客框架，KMblog 为开发者提供了完整的开发体验：

- **Vue 3 + Vite** - 现代化技术栈，开发体验丝滑
- **组件化架构** - 轻松扩展和自定义功能
- **完整测试** - 单元测试 + 属性测试，保证代码质量
- **CI/CD 集成** - 自动构建和发布，专注于内容创作

### 跨平台管理工具

管理工具不是网页应用，而是真正的桌面软件：

- **Windows / macOS / Linux** - 三大平台原生支持
- **离线工作** - 无需网络连接即可管理博客
- **自动更新** - 框架更新一键完成，无需手动操作
- **GitHub 集成** - 直接在工具中完成部署，无需命令行

## 快速开始

### 方式一：使用管理工具（推荐）

1. 从 [Releases](https://github.com/iiishop/KMblog/releases) 下载对应平台的管理工具
2. 解压并运行 `KMblogManager`
3. 点击"初始化博客"按钮
4. 开始创作你的第一篇文章

### 方式二：从源码开始

```bash
# 克隆仓库
git clone https://github.com/iiishop/KMblog.git
cd KMblog

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 运行管理工具
python blog_manager.py
```

## 配置说明

博客配置文件位于 `src/config.js`，也可以通过管理工具的"配置管理"界面进行可视化编辑。

主要配置项：

```javascript
{
  ProjectUrl: 'https://your-blog.github.io',  // 博客地址
  BlogName: '你的博客名称',
  ShortDesc: '博客简介',
  
  // 主题配置
  LightTheme: 'day',      // 浅色主题
  DarkTheme: 'dark',      // 深色主题
  defaultMode: 'system',  // 跟随系统
  
  // 背景设置
  BackgroundImg: '/assets/background.png',
  BackgroundImgBlur: 20.0,
  
  // 文章展示
  PostsPerPage: 5,
  EnableDynamicPostsPerPage: true,
  
  // 社交链接
  Links: [
    { name: 'GitHub', url: 'https://github.com/...' }
  ]
}
```

## 部署到 GitHub Pages

### 使用管理工具（推荐）

1. 打开管理工具
2. 点击"部署到 GitHub"
3. 输入 GitHub Token 和仓库名称
4. 点击"开始部署"

### 手动部署

```bash
# 构建项目
npm run build

# 部署到 gh-pages 分支
git subtree push --prefix dist origin gh-pages
```

## 开发指南

### 添加自定义组件

1. 在 `src/components/` 创建组件文件
2. 在 `src/config.js` 中注册组件名称
3. 将组件添加到对应的列表配置中

示例：

```javascript
// src/config.js
InfoListUp: [
  'SelfIntroductionPanel',
  'YourCustomPanel',  // 添加你的组件
],
```

### 自定义主题

编辑 `src/color.css` 添加新主题：

```css
[data-theme="your-theme"] {
  --theme-primary: #your-color;
  --theme-body-bg: #your-bg;
  --theme-body-text: #your-text;
  /* 更多变量... */
}
```

然后在 `src/config.js` 中配置：

```javascript
availableThemes: ['day', 'dark', 'your-theme'],
LightTheme: 'your-theme',
```

### 运行测试

```bash
# 运行所有测试
npm run test

# 生成覆盖率报告
npm run test:run

# 使用 UI 界面
npm run test:ui
```
